# ArchGuard Hygiene Rules

Code hygiene rules. These are not architectural invariants — they do not prevent production failures. They prevent code rot, maintain cognitive load within bounds, and enforce clean code practices. They live in the same `eslint.config.cjs` as the invariant bans but serve a different purpose.

---

## Rule Set

| # | Rule | Value | Justification |
|:---|:---|:---|:---|
| H1 | Cyclomatic Complexity | `max: 8` | **McCabe's Theorem (1976)** — M ≤ 10 is the upper bound for comprehensibility. Defect density increases sharply above 10. |
| H2 | Nesting Depth | `max: 3` | Pyramid of Doom avoidance. Beyond 3 levels, logic becomes impossible to reason about linearly. |
| H3 | Function Length | `max-lines-per-function: 75` (warn) | **Lipow (1982)** — defect density explodes after 100 lines. 75 is the empirical sweet spot. Exclude comments and blank lines. |
| H4 | Unused Variables | `error` | `@typescript-eslint/no-unused-vars` — dead code elimination. Cognitive load (Miller's Law). |
| H5 | Dead Code (cross-file) | `knip` in CI | `knip` flags orphaned exports, unused files, and unused dependencies. Runs separately from ESLint. |
| H6 | Risk Index | `< 60` | Formula: `(Cyclomatic Complexity × Lines of Code) / 10`. Validated by NASA/JPL. CC=8 × LOC=75 / 10 = 60. |

---

## ESLint Configuration

```js
// Merged into eslint.config.cjs by archguard-implement
{
  rules: {
    "complexity": ["error", { max: 8 }],
    "max-depth": ["error", { max: 3 }],
    "max-lines-per-function": ["warn", {
      max: 75,
      skipBlankLines: true,
      skipComments: true
    }],
    "@typescript-eslint/no-unused-vars": ["error", {
      argsIgnorePattern: "^_",
      varsIgnorePattern: "^_"
    }]
  }
}
```

## Knip (Dead Code Detection)

Run `knip` in CI. Not part of ESLint — a separate tool that understands cross-file references.

```json
// knip.json (generated by archguard-implement if knip is in devDependencies)
{
  "ignore": ["**/__tests__/**", "**/*.test.ts"],
  "ignoreDependencies": ["@types/*"]
}
```

Add to `package.json` scripts:
```json
{
  "scripts": {
    "knip": "knip"
  }
}
```

## When Rules Are Skipped

| Condition | Action |
|:---|:---|
| `complexity`, `max-depth`, `max-lines-per-function` | Always included |
| `no-unused-vars` | Always included (TS project) |
| `knip` | Included only if `knip` is in devDependencies |

---

## Why These Are Separate from Invariants

| Invariant Bans | Hygiene Rules |
|:---|:---|
| Prevent production failures | Prevent code rot |
| Block agent shortcuts that pass tests but break at runtime | Block code that works but is unmaintainable |
| Architectural (FM1-FM6) | Cognitive (Miller's Law, McCabe, Lipow) |
| Tests cannot catch these | Tests pass fine, but developers suffer |
